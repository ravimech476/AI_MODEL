How the Models Are Implemented & How to Use Them with Different Inputs
Big idea: all the model logic lives in ONE file — ml_core.py. Your team calls its functions (never rewrites them) and changes the file, target, columns, or model to try different inputs.

Part B — Helper Functions

numeric_columns(df) — find valid target columns

def numeric_columns(df):
    out = []
    for c in df.columns:
        if pd.to_numeric(df[c], errors="coerce").notna().sum() > 0:
            out.append(c)
    return out
→ Keeps only columns that contain numbers. Fills the target dropdown (you can only predict numeric columns).

suggest_leakage_columns(columns, target) — auto-find leakage

def suggest_leakage_columns(columns, target):
    return [c for c in columns
            if c != target and any(tok in c.upper() for tok in _LEAK_TOKENS)]
→ Returns disease columns (SEV, SC_MEAN, …) EXCEPT the target — so they get excluded automatically.

Part C — make_model() — build a model

def make_model(algorithm):
    if algorithm == "XGBoost":
        return XGBRegressor(n_estimators=600, learning_rate=0.03, max_depth=5, ...)
    if algorithm == "Random Forest":
        return RandomForestRegressor(n_estimators=300, ...)
→ A factory: give it a name string, it returns a fresh untrained model. This is how the code supports BOTH algorithms from one place.

Part D — prepare_features() — THE most important function

def prepare_features(df, target="SEVERITY", extra_drop=()):
    drop = set(BASE_DROP) | {target} | set(extra_drop)        # 1
    X = df.drop(columns=[c for c in drop if c in df.columns]) # 2
    for c in X.columns:                                        # 3
        if X[c].dtype == object:
            X[c] = X[c].astype("category").cat.codes           # 4
    X = X.apply(pd.to_numeric, errors="coerce").fillna(0)     # 5
    return X
LineWhat it does
1Build the "remove" list = ID columns + the target + your extra exclusions
2Drop those columns → what's left = the features
3–4Any text column (METHOD, SOIL) → convert to numbers (models need numbers)
5Force everything numeric, fill blanks with 0
This function makes "different inputs" work. Whatever Excel you give it, it auto-cleans it into model-ready numbers — your team never prepares data by hand.

Part E — train_all_variants() — TRAIN (the main button)

results = ml.train_all_variants(
    df,                                       # your data
    algorithms=["XGBoost","Random Forest"],   # which models
    target="SEV_Mean",                        # what to predict
    extra_drop=["Fungi","DAP"],               # what to exclude
    cv=True)                                  # cross-validate?
→ For each algorithm × each variant: calls prepare_features(), splits the data, trains the model, scores it. Returns a dictionary keyed by "Algorithm · Variant", each holding {model, metrics, importance, features, target}.

Part F — predict() — PREDICT (use the model)

def predict(bundle, df):
    model = bundle["model"]
    feats = bundle["features"]
    X = prepare_features(df, bundle["target"])      # clean the new data
    X = X.reindex(columns=feats, fill_value=0)      # match training columns exactly
    df["PREDICTED_SEVERITY"] = model.predict(X)     # the actual prediction
    return df
→ Key line: reindex(columns=feats) lines up the new data's columns to EXACTLY match what the model trained on. That is why prediction works even if the new file has extra or missing columns.

Part G — fit_lmm() — EXPLAIN (statistics)

table, summary, info = ml.fit_lmm(df, fixed=ml.LMM_FIXED, target="SEV_Mean")
→ Builds a formula "SEV_Mean ~ NDVI_MEAN + PSRI_MEAN + …", fits a mixed model grouped by POINT, and returns a table of coefficients + p-values (which signals significantly affect disease).

How It All Connects

YOUR INPUT (df, target, extra_drop) | prepare_features() <- cleans into numbers (used by everything) | make_model() <- builds XGBoost or Random Forest | train_all_variants() <- trains & scores -> results dict | predict() <- uses a trained model on new data fit_lmm() <- explains with statistics

How to Use It with DIFFERENT Inputs (real code)

import ml_core as ml
import pandas as pd

df = pd.read_excel("file.xlsx")                       # <- different FILE here

results = ml.train_all_variants(
    df,
    algorithms=["Random Forest"],                     # <- different MODEL here
    target="NEW_SC_Mean",                             # <- different TARGET here
    extra_drop=ml.suggest_leakage_columns(df.columns,"NEW_SC_Mean") + ["DAP"],  # <- different COLUMNS
    cv=True)

print(ml.comparison_table(results))                   # see the scores
