🌽 How to Use the Models with Different Inputs

A Simple Team Guide — Corn Disease AI

1. Think of the Model Like a Juice Machine 🥤

A juice machine: you put in fruit, it gives you juice.

You put in FRUIT   →   machine works   →   JUICE comes out

Our model works the same way:

You put in DRONE DATA   →   model works   →   DISEASE PREDICTION

"Using the model with different inputs" simply means putting in different data.

2. The 4 Things You Can Change

To try different inputs, change one (or more) of these:

Change thisExampleLike…
1. The fileUse a different Exceldifferent fruit
2. The targetPredict severity OR stroma countwhat juice you want
3. The columnsInclude or exclude Fungi / DAPwhich ingredients
4. The modelXGBoost or Random Forestwhich machine

3. Where Everything Lives

All the model code is in ONE file: ml_core.py

Think of ml_core.py as the juice machine's motor. You do not rebuild the motor — you just press buttons (call its functions).

4. The 3 Buttons (Functions) You Use

Button 1 — TRAIN (teach the model)

results = ml.train_all_variants(df, target="SEV_Mean")
In plain words: "Learn from this data, predict this column."

Button 2 — PREDICT (use the model)

predictions = ml.predict(model, new_data)
In plain words: "Look at this new data, tell me the disease."

Button 3 — EXPLAIN (statistics / LMM)

table = ml.fit_lmm(df, target="SEV_Mean")
In plain words: "Tell me which signals matter, with proof."

5. A Super-Simple Example (the whole thing)

import ml_core as ml          # turn on the machine
import pandas as pd

# 1. Put in the data (INPUT)
df = pd.read_excel("my_file.xlsx")

# 2. Press the TRAIN button — tell it what to predict
results = ml.train_all_variants(df, target="SEV_Mean")

# 3. See the result
print(ml.comparison_table(results))

To try a DIFFERENT input, change just ONE line:

df = pd.read_excel("DIFFERENT_file.xlsx")              # different file
# OR
results = ml.train_all_variants(df, target="NEW_SC_Mean")   # different target

6. The Key Message

You do NOT write new code. You change the INPUT (file, target, columns) and press the same buttons (functions). The machine handles the rest.

7. How to Learn It

Open the file use_models_examples.py. It has ready examples. Learn by doing:

  • RUN it once — see it work.
  • CHANGE one thing — e.g. the target column.
  • RUN it again — see what changes.
  • Repeat: change the file, the columns, the model.

8. Summary

All model logic is in ml_core.py. You use 3 functions — train_all_variants(), predict(), fit_lmm() — and change the file, target, columns, or model to try different inputs. You never rewrite the model; you just feed it different data. The tutorial file use_models_examples.py lets you learn by editing and re-running.

One-line takeaway: Same machine, different fruit. Change the input, press the same buttons.
